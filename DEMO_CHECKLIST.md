# UnitFlow AI MVP 데모 체크리스트

## 데모 전 준비

1. 백엔드 실행 확인
2. seed 데이터 적재 확인
3. `GET /health` 응답 확인
4. 프런트에서 API Base URL이 `http://localhost:8000` 기준인지 확인
5. 강사 계정, 학생 계정 로그인 정보 확인

## 실행 순서

1. 백엔드 실행

```bash
cd apps/api
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
copy ..\..\.env.example .env
python -m app.seed
uvicorn app.main:app --reload
```

2. 프런트 실행 환경 준비

- 실제 프런트 기준 파일: `apps/frontend/web.js`
- 브라우저 전역값 사용 시 예시:

```html
<script>
  window.__UNITFLOW_API_BASE_URL__ = "http://localhost:8000";
</script>
```

## 데모 시연 흐름

### 1. 강사용 흐름

1. 강사 계정 로그인
2. 강사용 대시보드 진입
3. 상담 우선 학생 확인
4. 학생 목록 진입
5. 특정 학생 상세 진입
6. 진단 근거, 전략 요약, 과목별 위치, 최근 시험 확인
7. 대학 정책 화면 진입
8. 대학별 반영 비율과 목표 점수 확인
9. 시험 관리 화면 진입
10. 시험명, 학원, 과목, 시험일, 총점 입력 후 시험 생성

### 2. 학생용 흐름

1. 학생 계정 로그인
2. 학생용 대시보드 진입
3. 오늘의 집중 포인트 확인
4. 전략 요약과 목표 대학 대비 위치 확인
5. 최근 점수와 목표 격차 확인

## 확인 포인트

- 취약 유형이 단순 점수 나열이 아니라 구조화된 진단으로 보이는지 확인
- 전략 설명이 목표 대학 기준과 최근 학습 데이터에 맞게 보이는지 확인
- 학생 비교나 낙인 중심 화면이 기본값이 아닌지 확인
- 시험 생성 시 하드코딩 값이 아니라 실제 메타데이터를 쓰는지 확인
- API 실패 시 fallback 화면이 깨지지 않는지 확인

## 데모 중 보여줄 핵심 메시지

- 본체는 단순 대시보드가 아니라 진단과 전략 생성 엔진
- 분석은 결정론적 구조 중심
- 전략 설명은 계산 결과를 넘지 않도록 제한
- 목표 대학 반영 규칙을 데이터로 설정 가능
- 강사용 화면과 학생용 화면이 같은 분석 엔진을 공유

## 현재 제약사항

- `apps/frontend/web.js` 안에 과거 중복 선언이 남아 있음
- 현재 렌더링은 마지막 활성 선언 기준으로 동작
- Python `3.14.x` 환경에서는 일부 패키지 설치 호환성 이슈 가능성 존재
- 전체 런타임 통합 검증은 환경에 따라 추가 확인 필요

## 데모 종료 후 점검

1. 생성한 시험이 목록 상단에 보이는지 확인
2. 로그인 세션 복원이 유지되는지 확인
3. 학생 상세 진입과 복귀 동선이 유지되는지 확인
4. API 오류 메시지가 화면을 깨뜨리지 않는지 확인
